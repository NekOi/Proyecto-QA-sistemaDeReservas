# Bugs Reportados – Sistema de Reservas Online

## Bug 1: El historial de reservas no muestra reservas existentes

**Precondición:**  
- Tener al menos una reserva confirmada  

**Pasos para reproducir:**  
1. Iniciar sesión  
2. Tener al menos una reserva confirmada  
3. Ir a “Mi historial de reservas”  

**Resultado esperado:**  
- Se muestran todas las reservas realizadas por el usuario, con fecha, hora y estado  

**Resultado actual:**  
- La lista aparece vacía aunque existen reservas registradas  

**Prioridad:** Alta  
**Entorno:** Chrome  
**Sistema operativo:** Windows 10  
**Versión del sistema:** v1.0  
**Notas:** El bug se registró al primer intento  

---

## Bug 2: No se envía correo de confirmación después de completar una reserva

**Precondición:**  
- Usuario registrado e iniciado sesión en el sistema  

**Pasos para reproducir:**  
1. Iniciar sesión en la aplicación  
2. Buscar disponibilidad de reserva  
3. Seleccionar fecha y hora disponibles  
4. Confirmar la reserva  
5. Revisar bandeja de entrada del correo registrado  

**Resultado esperado:**  
- La reserva se completa correctamente en el sistema  
- Se recibe un correo de confirmación con los detalles de la reserva  

**Resultado actual:**  
- No se recibe ningún correo de confirmación  

**Frecuencia:** 5 de 5 intentos  
**Prioridad:** Alta  
**Entorno:** Chrome  
**Sistema operativo:** Windows 10  
**Versión del sistema:** v1.0

# Proyecto QA – Sistema de Reservas Online (Simulado)

## Descripción
Este es un proyecto simulado de QA realizado sobre una aplicación de reservas online.  
El objetivo del proyecto es practicar la creación y gestión de historias de usuario, reporte de bugs y ejecución de pruebas funcionales dentro de un entorno ágil Scrum utilizando Jira.

## Mi rol
- QA Junior  
- Creación de historias de usuario y definición de criterios de aceptación  
- Reporte, seguimiento y cierre de bugs  
- Ejecución de pruebas funcionales manuales  
- Simulación de sprint planning, ejecución de sprint y sprint review

## Herramientas y metodología
- Jira para gestión de historias y bugs  
- Metodología ágil Scrum  
- Pruebas funcionales manuales  

## Contenido del repositorio
- `Historias-de-usuario.md` → Listado de todas las historias de usuario con criterios de aceptación  
- `Bugs-reportados.md` → Registro de los bugs detectados durante la simulación de pruebas  
- `Evidencia-simulacion.pdf` (opcional) → Capturas de Jira mostrando las historias y bugs cerrados  

## Resultados
- Simulación de 2 sprints completos  
- Bugs detectados, reportados, vinculados a historias y cerrados tras validación  
- Experiencia práctica documentada de QA Junior  

## Nota
Este proyecto es **simulado** con fines de aprendizaje y portafolio profesional.
